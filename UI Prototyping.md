## Hueristic Evaluation
- *Visibility of system status*
  - One of our application’s pages is a loading screen that tells the user how much the system is processing loaded information after certain actions such as logging in, creating a new profile and generating the optimal route to meet-up location.
- *Match system words to the real world*
  - We use simple terms that are easy to understand such as “Login” and “Match” to describe the actions. 
- *User control and freedom*
  - Users enable/disable certain features of the system in the settings page. For example, they can determine how long they want to disable their location for. We also provide accessibility settings, such as text size, language options, and voice overs.
- *Consistency and standards*
  - We are implementing simple and recognizable icons for the user’s ease of navigation experience, such as arrows (next/back a page) and a cog wheel (settings page). We also decided on a simple color scheme that we are implementing throughout the pages of the application.
- *Error prevention*
  - Our system will notify the user if they input invalid credentials when logging in. Invalid email formats or invalid passwords will be detected before creating a new profile. The system will also have a general error message for any system crashes/errors.
- *Recognition rather than recall*
  - Standard icons/buttons are helpful for navigation and guidance, as we are using icons that have been used in other apps, and are therefore easily recognizable.
- *Accelerators*
  - For efficient navigation, users can simply swipe left and right to navigate the pages, rather than clicking the arrow icons.
- *Minimalist design*
  - The system’s design is not overshadowed by overstimulating visuals, colors, and images; the pages maintain consistent colors and shapes throughout the app.
- *Help users recognize and recover from errors*
  - Error messages/pages will pop up when there is a system-side error. The user will be given helpful suggestions to overcome the error, such as restarting the app or logging off and logging back in.
- *Help and documentation*
  - There will be a question mark button on each page to help the user navigate the app. A simple dialog box will pop up near the question mark once clicked, explaining to the user the features of the page.
